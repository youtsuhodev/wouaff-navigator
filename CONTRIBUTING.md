# Contribuer à Wouaff

Merci de votre intérêt pour contribuer à Wouaff ! 🐾

## Comment Contribuer

### 1. Signaler un Bug

Ouvrez une [Issue](https://github.com/youtsuhodev/wouaff-navigator/issues) avec :

- **Titre clair** décrivant le problème
- **Étapes pour reproduire** le bug
- **Comportement attendu** vs comportement actuel
- **Capture d'écran** si applicable
- **Version** de Wouaff utilisée
- **Système d'exploitation**

### 2. Proposer une Feature

Ouvrez une [Issue](https://github.com/youtsuhodev/wouaff-navigator/issues) avec le label `enhancement` :

- **Description** de la fonctionnalité
- **Pourquoi** elle serait utile
- **Comment** vous l'imagineriez

### 3. Envoyer une Pull Request

```bash
# 1. Fork le projet
git clone https://github.com/VOTRE_USERNAME/wouaff-navigator.git

# 2. Créer une branche
git checkout -b feature/ma-feature

# 3. Faire vos modifications

# 4. Tester localement
npm run dev

# 5. Commit
git commit -m "feat: ajout de ma feature"

# 6. Push
git push origin feature/ma-feature

# 7. Ouvrir une Pull Request
```

## Règles

- **Nom des branches** : `feature/xxx`, `fix/xxx`, `docs/xxx`
- **Messages de commit** : suivre [Conventional Commits](https://www.conventionalcommits.org/)
- **Code** : respecter le style existant
- **Tests** : ajouter des tests si nécessaire
- **Documentation** : mettre à jour le README si nécessaire

## Style de Code

- 2 espaces pour l'indentation
- Points-virgules en fin de ligne
- Noms de variables en camelCase
- Fonctions en camelCase
- Constantes en UPPER_SNAKE_CASE
